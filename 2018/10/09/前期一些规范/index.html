1 前言

以下是项目的目录架构和代码的书写规范，在这学期内，项目的命名规范按照这个目录架构来，而代码的书写规范则一直使用以下规范。
本文档的目标是使前端代码风格保持一致，容易被理解和被维护。大家要严格遵守这个规范哦。

2 项目目录架构

README.md

每个项目都必须「MUST」包含一个README.md文件，此文件中应当「SHOULD」概要描述此项目的功能和特点等信息。

src

项目中所有 JS 源码应当「SHOULD」存放在此目录下，且所有JS文件编写应当「SHOULD」遵循Javascript 编码规范。

css、themes、less、sass

样式类文件存放应当「SHOULD」遵循以下规律，且文件编写应当「SHOULD」遵循CSS 编码规范。

不带主题的样式文件应当「SHOULD」统一存放在 css 目录下面，且样式中使用的背景图片资源应当「SHOULD」统一存放在 css/images 目录下面。
带主题的样式文件应当「SHOULD」统一存放在 themes 目录下对应的主题目录下，默认的主题应当「SHOULD」采用 default 作为主题名称，且应当「SHOULD」默认提供，样式中对应图片文件应当「SHOULD」存放在样式文件所在的主题目录下的 images 目录下。
less 格式的样式文件应当「SHOULD」统一存放在 less 目录下面。
sass 格式的样式文件应当「SHOULD」统一存放在 sass 目录下面。

doc

所有项目应当「SHOULD」包含一个 doc 目录，用来存放详细的 API 使用文档。

dist

dist 作为项目输出目录，所有编译生成、提供给用户使用的文件应当「SHOULD」存放在此目录。
为了让不太擅长 node.js 的用户可以正常使用编译后的代码，dist 目录应当「SHOULD」包含基本输出结果并提交在 github 中。

build

所有工具类脚本应当「SHOULD」放在此目录。

test

所有测试相关代码应当「SHOULD」放在此目录。

##src_cov

为了测试代码覆盖率，所有为测试覆盖率生成的新 JS 文件应当「SHOULD」存放在此目录下面。

3 前端规范

3.1 HTML规范

3.1.1 代码风格

3.1.1.1 缩进和换行

建议使用Tab作为一个缩进层级,设置Tab为4个空格。

3.1.1.2 命名

  1. id和class必须单词全部字母小写，单词间以_分割。
  2. 元素id必须保证页面唯一，必须单词全部字母小写，单词间以_分割。

解释：

同一个页面中，不同的元素包含相同的 id，不符合 id 的属性含义。并且使用 document.getElementById 时可能导致难以追查的问题。

3.1.1.3 标签

  1. 标签名必须使用小写字母。
  2. 对于无需自闭合的标签，不允许自闭合。
  3. HTML标签的使用应该遵循标签的语义化。

3.1.1.4 属性

  1. 属性名必须使用小写字母。
  2. 属性值必须用双引号包围。
  3. 布尔类型的属性，建议不添加属性值。
  4. 自定义属性建议以 xxx- 为前缀，推荐使用 data-

解释：

  1. 不允许使用单引号，不允许不使用引号。
  2. 使用前缀有助于区分自定义属性和标准定义的属性。

3.1.2 通用

3.1.2.1 DOCTYPE

使用 HTML5 的 doctype 来启用标准模式，建议使用大写的 DOCTYPE。

3.1.2.2 设置编码

明确指定字符编码。指定字符编码的 meta 必须是 head 的第一个直接子元素。

3.1.2.3 文件引入

  1. 引入CSS时必须指明rel="stylesheet"。
  2. 引入 CSS 和 JavaScript 时无须指明 type 属性。
  3. 在 head 中引入页面需要的所有 CSS 资源。
  4. JavaScript 应当放在页面末尾，或采用异步加载。

解释：

  1. 结构-样式-行为的代码分离，对于提高代码的可阅读性和维护性都有好处。
  2. 在页面渲染的过程中，新的CSS可能导致元素的样式重新计算和绘制，页面闪烁。
  3. 将script放在页面中间将阻断页面的渲染。出于性能方面的考虑，如非必要，请遵守此条建议。

3.1.3 图片

  1. 禁止给img的src取值为空。
  2. 避免给img添加不必要的title属性。
  3. 为重要的图片添加alt属性。
  4. 有下载需求的图片采用 img 标签实现，无下载需求的图片采用 CSS 背景图实现。

3.1.4. 表单

3.1.4.1 控件标题

有文本标题的控件必须使用 label 标签将其与其标题相关联。

解释：

有两种方式：

  1. 将控件置于label内。
  2. label的for属性指向控件的id。
    推荐使用第一种，减少不必要的id。如果DOM结构不允许直接嵌套，则应使用第二种。

3.1.4.2 按钮

  1. 使用 button 元素时必须指明 type 属性值。
  2. 尽量不要使用按钮类元素的 name 属性。

3.2 CSS规范

3.2.1 代码风格

3.2.1.1 缩进和换行

建议使用Tab作为一个缩进层级，设置Tab为4个空格。

3.2.1.2 空格

  1. 选择器 与 { 之间必须包含空格。
  2. 属性名和之后的:之间不能有空格，:和属性值之间有一个空格。

3.2.1.3 行长度

  1. 每行不得超过120个字符，除非单行不可分割。
  2. 对于超长的样式，在样式值的空格处或,后换行，建议按逻辑分组。

3.2.1.4 选择器

当一个 rule 包含多个 selector 时，每个选择器声明必须独占一行。

3.2.1.5 属性定义

  1. 属性定义必须另起一行。
  2. 属性定义后必须以分号结尾。

3.2.2 通用

3.2.2.1 属性缩写

在可以使用缩写的情况下，尽量使用属性缩写。

3.2.2.2 属性书写顺序

同一 rule set 下的属性在书写时，应按功能进行分组，并以 Formatting Model（布局方式、位置） > Box Model（尺寸） > Typographic（文本相关） > Visual（视觉效果） 的顺序书写，以提高代码的可读性。

3.2.2.3 !important

尽量不使用 !important 声明。

3.2.2.4 z-index

  1. 将 z-index 进行分层，对文档流外绝对定位元素的视觉层级关系进行管理。
  2. 在可控环境下，期望显示在最上层的元素，z-index 指定为 999999。
  3. 在第三方环境下，期望显示在最上层的元素，通过标签内联和 !important，将 z-index 指定为 2147483647。

3.2.3 值和单位

3.2.3.1 文本

文本内容必须用双引号包围。

3.2.3.2 数值

当数值为 0 - 1 之间的小数时，省略整数部分的 0。

3.2.3.3 url

  1. url() 函数中的路径不加引号。
  2. url() 函数中的绝对路径可省去协议名。

3.3 JavaScript规范

3.3.1 代码风格

3.3.1.1 缩进和换行

建议使用Tab作为一个缩进层级，设置Tab为4个空格。

3.3.1.2 空格

  1. 二元运算符两侧必须有一个空格，一元运算符与操作对象之间不允许有空格。
  2. 用作代码块起始的左花括号{前必须有一个空格。
  3. , 和 ; 前不允许有空格。
  4. 单行声明的数组与对象，如果包含元素，{} 和 [] 内紧贴括号部分不允许包含空格。
  5. 行尾不得有多余的空格。

3.2.1.3 换行

  1. 每个独立语句结束后必须换行。
  2. 每行不得超过 120 个字符。
  3. 运算符处换行时，运算符必须在新行的行首。
  4. 语句超过120行的时候要根据逻辑条件合理缩进。

3.2.1.4 命名

  1. 变量使用_下划线命名法
  2. 常量使用全部字母大写，单词间下划线分割的命名方式。
  3. 函数使用Camel命名法。

3.2.1.5 注释

  1. 单行注释：//
  2. 多行注释： /**/
  3. 文件顶部必须包含文件注释，用 @file 标识文件说明。

3.2.2 通用

3.2.2.1 条件

  1. 在 Equality Expression 中使用类型严格的===。仅当判断 null 或 undefined 时，允许使用== null。
  2. 尽可能使用简洁的表达式。

解释：
使用===可以判断中隐式的类型转换。

3.2.2.2 循环

不要在循环体中包含函数表达式，事先将函数提取到循环体外。

解释：

循环体中的函数表达式，运行过程中会生成循环次数个函数对象。

3.2.2.3 类型

类型检测优先使用typeof。对象类型检测使用instanceof。null或undefined的检测使用== null。

3.2.2.4 字符串

字符串开头和结束使用单引号 '。

解释：

  1. 输入单引号不需要按住shift，方便输入。
  2. 实际使用中，字符串经常用来拼接 HTML。为方便HTML中包含双引号而不需要转义写法。

3.2.2.5 对象

  1. 使用对象字面量{}创建新Object。
  2. 属性访问时，尽量使用.。

3.2.2.6 数组

  1. 使用数组字面量 [] 创建新数组，除非想要创建的是指定长度的数组。
  2. 遍历数组不使用 for in。

3.2.3 减少全局变量
